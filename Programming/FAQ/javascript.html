What data types exist in JavaScript?

Primitive Types (immutable, stored by value)

  • undefined ‚Äì default for uninitialized variables.
  • null ‚Äì intentional absence of any value.
  • boolean - true or false.
  • number - integers and floats (-100, 3.14).
  • bigint - for numbers larger than Number.MAX_SAFE_INTEGER.
  • string - text ("hello").
  • symbol - each Symbol() is guaranteed to be unique.

Non-Primitive Types (objects, stored by reference)

  • object - collections of key-value pairs ({ name: "Alice" }).
  • function - callable objects (() => {}), technically a subtype of object.
  • array - list-like objects (([1, 2, 3]), also objects.
  • date, regex, map, set - all object types too.
What is the Event Loop in JavaScript?

JavaScript runs in a single thread. The event loop is what makes JavaScript non-blocking and able to handle asynchronous code like setTimeout, fetch, or event listeners.

‚≠ê Even though the timeout is 0ms, it runs after the synchronous code, because it waits in the event queue.

setTimeout(() => console.log("Async"), 0);
console.log("End");

What is the Web APIs environment?

The Web APIs environment refers to the set of features provided by the browser (or Node.js), not JavaScript itself, that handle things like:

  • setTimeout, setInterval
  • DOM events (click, keydown, etc.)
  • fetch, XMLHttpRequest
  • console.log
  • WebSockets, geolocation, File API, etc.

‚≠ê These APIs run outside the JavaScript engine ‚Äî usually written in C++ inside the browser or Node.js.

Microtasks vs Macrotasks in JavaScript

  • Macrotasks - things like setTimeout, setInterval, setImmediate, and I/O callbacks.

  • Microtasks - things like Promise.then(), queueMicrotask(), MutationObserver.

  1. Runs all synchronous code first (top-level call stack).
  2. Then:
    • Flushes all microtasks (Promise.then, queueMicrotask, etc.)
  3. Then:
    • Runs one macrotask (e.g. setTimeout, setInterval)
    • After it finishes, flushes all microtasks again
  4. Repeats from step 3.

‚≠ê Even though both are async, the Promise (microtask) runs before the setTimeout (macrotask).

console.log('Start');

setTimeout(() => {
  console.log('Macrotask - setTimeout');
}, 0);

Promise.resolve().then(() => {
  console.log('Microtask - Promise');
});

console.log('End');

// Start
// End
// Microtask - Promise
// Macrotask - setTimeout
What is "This" Keyword in Javascript?

The this keyword refers to the execution context ‚Äî meaning who is calling the function.

‚≠ê TIts value depends on how the function is called

console.log(this); // In browser: Window (global object)

function show() {
  console.log(this);
}
show(); // Still Window (non-strict mode)

const obj = {
  name: 'Alice',
  greet() {
    console.log(this.name);
  }
};
obj.greet(); // Logs "Alice" ‚Äî because 'this' refers to obj
this in arrow functions vs regular functions 
function Timer() {
  this.seconds = 0;

  // ‚ùå non-strict mode -> 'this' points to the global object
  // ‚ùå strict mode -> 'this' is undefined
  setInterval(function () {
    this.seconds++;
    console.log("Regular:", this.seconds); // undefined or NaN
  }, 1000);

  // ‚úÖ Arrow function ‚Äî 'this' is inherited from Timer
  setInterval(() => {
    this.seconds++;
    console.log("Arrow:", this.seconds); // 1, 2, 3, ...
  }, 1000);
}

new Timer();
What is Promise?

A Promise is a JavaScript object that represents the future result of an asynchronous operation ‚Äî it may succeed (resolve), fail (reject), or still be pending.

‚≠ê Promise = a placeholder for a future value.

const promise = new Promise((resolve, reject) => {
  const success = true; // Simulate success or failure

  if (success) resolve("‚úÖ Success");
  else reject("‚ùå Error");
});

promise
  .then(result => console.log(result)) // "‚úÖ Success" if resolved
  .catch(error => console.log(error)); // "‚ùå Error" if rejected
What is async/await?

async and await are syntax built on top of Promises that make asynchronous code easier to read and write ‚Äî like synchronous code.

What are Generators?

Generator - function that stops and resumes like a series of checkpoints.

function* generatorFunc() {
  yield 1;
  yield 2;
  yield 3;
}

const gen = generatorFunc();
console.log(gen.next()); // { value: 1, done: false }
console.log(gen.next()); // { value: 2, done: false }
console.log(gen.next()); // { value: 3, done: false }
console.log(gen.next()); // { value: undefined, done: true }
How does the JS transpilation process work?

Transpilation is the process of converting modern JavaScript (e.g. ES6+) into an older, compatible version (like ES5) so it runs in older browsers.

What is closure?

A closure is a function remembers its outer variables even if it‚Äôs called later.

‚≠ê Closure = mini-class with hidden memory.

function outer() {
  let counter = 0;
  return function inner() {
    counter++;
    console.log(counter);
  };
}

const count = outer(); // outer() runs, returns inner()
count(); // 1
count(); // 2
What is an Object Prototype in JavaScript?

Prototype in JS ‚âà base class in C#, but more flexible and dynamic.

When you access a property or method like obj.something, JavaScript:

  1. Looks for something on obj itself.
  2. If not found, it looks at obj.__proto__ (its prototype).
  3. If still not found, it looks at the prototype‚Äôs prototype.
  4. ...and so on until it reaches null.
