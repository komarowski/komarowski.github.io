HTTP

What is HTTP?

HTTP (Hypertext Transfer Protocol) is the protocol used for communication between a client (like a browser) and a web server.

üß† Stateless = each request starts from scratch.

What is the difference between GET and POST requests?

  • GET - fetches data

    • Data in URL, visible
    • Can be cached, bookmarked, faster
    • ‚úÖ Safe to repeat

  • POST

    • Data in body, hidden
    • ‚ùå Not cached/bookmarked

Why use PUT and DELETE instead of just POST?

  • Makes your API self-explanatory. You know what the request is doing just by looking at it

  • Easier to test or integrate with tools (browsers, Postman, REST clients)

What is CORS (Cross-Origin Resource Sharing)?

CORS is a security feature in browsers that controls which websites can access resources from another domain using JavaScript.

By default, browsers follow the same-origin policy.

HTML

How the browser loads resources?

When a browser loads a webpage, it follows this general order:

  1. HTML is parsed first ‚Äî the browser reads the document top to bottom.

  2. CSS blocks rendering ‚Äî page can't be painted until all CSS is loaded and parsed.

  3. JavaScript can block HTML parsing unless marked with defer or async.

  4. Images, fonts, and other assets load later, usually in parallel unless marked as critical (e.g., via preload).

How <script> affects loading? 
<!-- Bad: blocks parsing -->
<!-- At the bottom of <body>, default behaves like defer -->
<script src="a.js"></script>

<!-- Better: downloads in parallel, runs after DOM ready -->
<!-- If multiple defer scripts are used, they execute in order -->
<script src="b.js" defer></script>

<!-- For non-critical analytics -->
<!-- Script is downloaded in parallel with HTML parsing -->
<script src="c.js" async></script>

CSS

What is specificity in CSS and how is it calculated?

Specificity determines which CSS rule wins when multiple rules apply.

  1. Inline styles - specificity value 1000

  2. ID selector #id - specificity value 100

  3. Class .class - specificity value 10

  4. Element div, h1 - specificity value 1

üß† More specific rules override less specific ones ‚Äî unless !important is used.

What‚Äôs the difference between em, rem, and px?

  • px - fixed pixels (absolute size)

  • em - relative to the font size of the parent

  • rem - relative to the root (html) font size

üß† Use rem for consistent scaling, em for nested flexibility, px when exact control is needed

What is the difference between visibility: hidden and display: none?

  • visibility: hidden - element is invisible but still takes up space

  • display: none - element is removed from the layout entirely

Why do we need pseudo-elements in CSS?

Pseudo-elements (like ::before and ::after) let you insert and style content in the page without adding extra HTML.

Networking concepts

How does your device get an IP address when you connect to the internet?

  • Your router gives your device a private IP.

  • Your Internet Service Provider (internet company) gives your router a public IP.

  • The router uses NAT (Network Address Translation) to connect your private IPs to the public internet.

Can two people in different homes have the same IP address?

They can have the same private IP (like 192.168.0.10) behind different routers but their public IPs must be unique on the internet.

This uniqueness is managed by ISPs and the global IP allocation system.

What is DNS?

DNS (Domain Name System) translates human-readable domain names (like google.com) into IP addresses (like 142.250.74.206) so browsers can load websites.

üß† DNS = domain name to IP translator

What is a CDN?

üß† CDN (Content Delivery Network) = global delivery system that brings your content closer to your users.
